API 연결 오류\n```\nFailed to fetch\n```\n**해결방법:**\n- 백엔드 서버 실행 상태 확인\n- CORS 설정 확인\n- 환경변수 NEXT_PUBLIC_BACKEND_URL 확인\n\n#### 2. 의존성 설치 오류\n**해결방법:**\n```bash\nrm -rf node_modules package-lock.json\nnpm install\n```\n\n## 📋 개발 중 체크리스트\n\n### 매일 개발 시작 전\n- [ ] MySQL 서버 실행 상태 확인\n- [ ] 가상환경 활성화 (백엔드)\n- [ ] Git pull로 최신 코드 동기화\n\n### 코드 수정 후\n- [ ] 백엔드 서버 재시작\n- [ ] 프론트엔드 hot reload 확인\n- [ ] API 응답 형식 확인\n- [ ] 브라우저 콘솔 에러 확인\n\n### 새 기능 추가 시\n- [ ] ERD 업데이트 필요성 확인\n- [ ] API 스키마 정의\n- [ ] 프론트엔드 타입 정의\n- [ ] SOLID 원칙 준수 확인\n\n## 🔧 추가 개발 도구\n\n### 데이터베이스 관리\n```bash\n# 마이그레이션 생성\ncd backend\nalembic revision --autogenerate -m \"Add new feature\"\n\n# 마이그레이션 적용\nalembic upgrade head\n```\n\n### API 테스트\n```bash\n# 간단한 서버 테스트\npython quick_test.py\n\n# 전체 서버 테스트\npython test_server.py\n```\n\n### 코드 품질 확인\n```bash\n# Python 코드 포맷팅\ncd backend\nblack .\nflake8 .\n\n# TypeScript 타입 확인\ncd frontend\nnpm run type-check\nnpm run lint\n```\n\n## 📚 참고 자료\n\n- **설계문서**: design-documents/ 디렉토리\n- **API 문서**: http://localhost:8000/docs (서버 실행 후)\n- **프로젝트 구조**: 각 디렉토리의 README.md 파일\n\n---\n\n**⚠️ 중요:** 이 가이드는 현재 코드 수정 상태를 반영합니다. 추가 수정 시 가이드도 함께 업데이트하세요.\n\n**📞 지원:** 문제 발생 시 GitHub Issues를 통해 문의하거나 팀 채팅으로 연락하세요.\n
